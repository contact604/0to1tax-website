**핵심 개선**: JavaScript를 이용한 불필요한 '정리' '설계' 텍스트 자동 제거

#### 주요 변경사항

**JavaScript 클린업 스크립트 추가**:
```javascript
document.addEventListener('DOMContentLoaded', function() {
    const allParagraphs = document.querySelectorAll('p');
    allParagraphs.forEach(p => {
        if (p.textContent.includes('정리') && p.textContent.includes('설계') && p.textContent.trim().length < 20) {
            p.style.display = 'none';
        }
    });
});
```

**동작 방식**:
- 페이지 로드 시 모든 `<p>` 태그 검사
- '정리'와 '설계'를 동시에 포함하고 길이가 20자 미만인 요소 찾기
- 해당 요소를 `display: none`으로 숨김

**2. 서브타이틀 강화** 💬

**기존**:
```
평소에 재무를 관리해 두면,
투자·실사·감사 요청이 와도 추가 준비가 필요 없습니다.
```

**개선**:
```
숫자를 '만드는 것'이 아니라,
대표가 판단할 수 있는 상태로 '관리'합니다.

평소 재무가 정리·설명·공유되어 있다면
투자·실사·감사 요청이 와도
급하게 준비할 일은 없습니다.
```

**핵심**:
- ✅ '만드는 것' → '관리' (톤 변경)
- ✅ '대표가 판단할 수 있는 상태' (목표 명확화)
- ✅ '정리·설명·공유' (3가지 핵심 행동)

---

**3. 4단계 업무 방식 재구성** 🔄

#### Step 1: 숫자는 정확하게, 하지만 '결정 기준'으로 관리합니다

**PROBLEM 연결**: 💬 숫자는 있지만 결정 불가

**내용**:
```
정확한 기장과 세무 신고를 기본으로 하되,
매출·비용을 대표·투자자·감사인이 보는 기준에 맞춰 관리합니다.

✓ 단순 장부 기록이 아니라 "왜 이렇게 나왔는지" 설명 가능한 구조로 관리
✓ 기장 결과가 대표의 사업 판단과 연결되도록 정리

👉 기장은 시작일 뿐, 목표는 의사결정입니다.
```

---

#### Step 2: 분기마다 투자 전문 회계사가 직접 설명합니다

**PROBLEM 연결**: 💬 설명·커뮤니케이션 부재

**내용**:
```
자동 리포트가 아니라,
투자·실사 경험을 가진 회계사가
숫자를 해석하고 연결해 설명합니다.

✓ 분기별 재무 상태 핵심 요약
✓ IR 스토리와 숫자의 연결 여부 점검
✓ 구조적 리스크 사전 식별
✓ 다음 분기에 준비해야 할 포인트 정리

👉 "보고서"가 아니라, 대표를 위한 분기 점검입니다.
```

---

#### Step 3: 업종 비교 + 현금흐름으로 회사 상태를 점검합니다

**PROBLEM 연결**: 📊 회사가 잘 가는지 스스로 판단 불가

**내용**:
```
절대 숫자만 보지 않습니다.

✓ 동종 업종 대비 매출 구조·비용 구조 비교
✓ 현금이 잘 돌고 있는지, 구조적 문제는 없는지 점검
✓ 세무 리스크를 사전에 관리해 현금 유출 가능성 최소화

👉 이 회사가 지금 잘 가고 있는지, 망할 위험은 없는지 분기마다 점검합니다.
```

```
✓ 감사 대응
✓ PA, 기업가치 평가, IFRS Conversion 등 확장 업무 연계

👉 투자 시점에 구조를 바꾸는 게 아니라, 이미 준비된 상태로 들어갑니다.
```

```
랜딩 페이지 (/)         → 문의 전환 전용
    ↓
/insights/              → 투자 전문 콘텐츠 (전환 보조)
    ↓
/blog/                  → 일반 콘텐츠 (검색 유입)
```

---

#### 1. **사이트 구조 재설계** ✅

**새로운 디렉토리 구조**:
```
0to1tax.com/
├─ /                    : 메인 랜딩 (문의 전환 전용)
├─ /insights/           : 투자 준비 인사이트 (PRE-A ~ Series A 전문)
│  ├─ index.html        : 인사이트 목록 (필터 기능)
│  ├─ ir-numbers.html   : VC 미팅 재무 숫자 (완성)
│  └─ ...               : 추가 콘텐츠 (템플릿)
└─ /blog/               : 일반 회계·보조금 블로그
   ├─ index.html        : 블로그 목록 (필터 기능)
   └─ ...               : 일반 콘텐츠 (템플릿)
```

---

#### 2. **/insights/ 설계 원칙** ✅

**콘텐츠 기준**:
- **주제**: 투자 / IR / 실사(FDD) / 재무 구조 / IFRS
- **톤**: 판단을 돕는 가이드 (설명형 블로그 ❌)
- **대상**: PRE-A ~ Series A 투자 준비 중인 대표급

**랜딩 연계**:
- ✅ 메인 랜딩 하단에 3개만 노출
- ✅ "투자 준비 인사이트" 명칭 사용 (블로그 ❌)
- ✅ 각 글 하단에 CTA: "투자 준비 진단 요청하기" → 랜딩 문의 폼

**디자인**:
- 파란색 그라데이션 (투자 전문성 강조)
- 태그: IR·미팅, 실사 대응, 법인·세무 구조 등

---

#### 3. **/blog/ 설계 원칙** ✅

**콘텐츠 기준**:
- **주제**: 보조금, 세무 상식, 회계 일반, 스타트업 팁
- **목적**: 검색 유입(SEO) 및 정보 제공
- **대상**: 초기 스타트업, 일반 사업자 포함

**중요한 제한**:
- ❌ 메인 랜딩 하단에 노출하지 않음
- ❌ 투자 전환용 CTA 강하게 넣지 않음
- ✅ 푸터에서만 접근 가능

**디자인**:
- 녹색 그라데이션 (일반 정보 느낌)
- 태그: 보조금, 세무 상식, 회계 일반, 스타트업 팁

---

#### 4. **메뉴 및 노출 원칙** ✅

**상단 네비게이션** (Fixed):
- ✅ "Zero to One TAX" 로고 (홈 링크)
- ✅ "투자 준비 인사이트" (insights 우선 노출)

1. **문의 허들 낮추기 섹션** ✅

## 🔌 블로그 시스템 작동 원리

```

   ↓

   ↓
3. GitHub Webhook → Netlify 자동 배포
   ↓
4. 사용자 웹사이트 접속
   ↓
5. JavaScript가 GitHub API로 Markdown 읽기
   ↓
6. Marked.js로 HTML 변환 → 화면 표시

// 게시글 작성
POST /tables/blog_posts

// 게시글 수정

// 게시글 삭제
DELETE /tables/blog_posts/{id}

// 상담문의 저장
POST /tables/consultations

// 상담문의 목록
GET /tables/consultations?sort=-created_at&limit=100
```

### Version 6.28.0 (2025-10-31) 📑 Modal Tabs UI!

```
┌─────────────────────────────┐
│  [기본 서비스] [추가 서비스]  │  ← 탭 버튼
├─────────────────────────────┤
│  18만원/월 (VAT 별도)        │
│  엔터프라이즈 플랜 포함       │
│                              │
│  ✓ 9개 서비스               │
└─────────────────────────────┘

탭 전환 시:
[기본 서비스] → 엔터프라이즈 9개 서비스
[추가 서비스] → 그로스 전용 5개 서비스
```

**스케일업 플랜 모달:**
```
┌─────────────────────────────┐
│  [기본 서비스] [추가 서비스]  │  ← 탭 버튼
├─────────────────────────────┤
│  18만원/월 (VAT 별도)        │
│  엔터프라이즈 플랜 포함       │
│                              │
│  ✓ 9개 서비스               │
└─────────────────────────────┘

탭 전환 시:
[기본 서비스] → 엔터프라이즈 9개 서비스
[추가 서비스] → 스케일업 전용 4개 서비스
```

**탭 디자인:**
- 활성 탭: 밝은 배경 + 진한 텍스트
  - 그로스: 화이트 배경 + 퍼플 텍스트

- 부드러운 전환 애니메이션

**JavaScript 함수:**
- `switchGrowthTab('basic' | 'additional')` - 그로스 플랜 탭 전환
- `switchScaleupTab('basic' | 'additional')` - 스케일업 플랜 탭 전환

**개선 효과:**
- ✅ 모달 높이 50% 감소
- ✅ 모바일 스크롤 최소화
- ✅ 한 화면에 모든 정보 표시
- ✅ 깔끔한 UI/UX
- ✅ 정보 탐색 용이

**사용자 경험:**
1. 모달 오픈 → 기본 서비스 탭 자동 선택
