# 📧 이메일 연동되어 있을 때 도메인 연결하기

## 🎯 상황
- ✅ 이메일: `누군가@0to1tax.com` 이미 사용 중
- ✅ 도메인: 0to1tax.com 구매 완료
- ❌ 웹사이트: 아직 연결 안 됨
- 🤔 문제: 이메일 망가질까 걱정

---

## 😊 걱정 마세요!

**이메일은 안전해요!**

이메일과 웹사이트는 **별개**예요:
- 이메일: **MX 레코드** (메일 서버)
- 웹사이트: **A 레코드** 또는 **CNAME 레코드** (웹 서버)

→ 웹사이트 연결해도 이메일은 그대로 작동! ✅

---

## 🔍 먼저: 현재 설정 확인하기

### 도메인이 어디서 관리되나요?

#### Case 1: Squarespace에서 구매 (가능성 높음)
→ Squarespace DNS 설정 화면에서 관리

#### Case 2: Google Workspace 사용 중
→ Google Admin Console에서 관리

#### Case 3: 기타 (GoDaddy, Gabia 등)
→ 해당 사이트 로그인

---

## ✅ 가장 쉬운 방법 (이메일 안전!)

### 🔵 방법: CNAME 레코드만 추가하기

**장점**:
- ✅ 이메일 영향 없음 (100% 안전!)
- ✅ 설정 간단 (5분)
- ✅ 빠르게 적용 (15분~2시간)

**단점**:
- www.0to1tax.com만 작동 (루트는 나중에)

---

## 📝 단계별 실행 (5분!)

### Step 1: Squarespace 로그인

1. **https://www.squarespace.com** 접속
2. 로그인

### Step 2: DNS 설정 열기

1. **도메인** 메뉴 클릭
2. **0to1tax.com** 선택
3. **DNS 설정** 클릭

### Step 3: 현재 레코드 확인

**확인할 것**: MX 레코드가 있는지 확인
```
Type: MX
Host: @
Points to: aspmx.l.google.com (또는 비슷한 것)
```

**있으면**: ✅ 이메일 설정이 있음 (건드리지 마세요!)
**없으면**: 이메일 설정 확인 필요

### Step 4: CNAME 레코드 추가

**중요**: 기존 레코드는 **절대 삭제하지 마세요!**

**추가할 레코드**:
```
Type: CNAME
Host: www
Points to: [배포된 URL에서 https:// 빼고 입력]
TTL: Auto 또는 3600
```

**예시**:
```
배포 URL이 https://iadkczem.gensparkspace.com 이라면
→ Points to: iadkczem.gensparkspace.com
```

### Step 5: 저장

**"추가" 또는 "저장"** 버튼 클릭

### Step 6: 대기 (15분~2시간)

커피 한 잔 마시고 기다리세요! ☕

### Step 7: 테스트

브라우저에서 접속:
```
https://www.0to1tax.com
```

**작동하면**: ✅ 성공!
**안 되면**: 조금 더 기다리기 (최대 2시간)

## 🎯 배포된 URL 찾는 법

### Publish 탭에서 확인:

1. 왼쪽 사이드바 **"Publish"** 탭 클릭
2. **"Deployed URL"** 또는 **"Live Site"** 항목 찾기
3. URL 복사

**형식**:
```
https://[프로젝트ID].gensparkspace.com
또는
https://iadkczem.gensparkspace.com (예시)
```

## ❓ 자주 묻는 질문

### Q1: 이메일이 망가질까요?
**A:** 아니요! MX 레코드를 건드리지 않으면 100% 안전해요.

**확인 방법**:
- CNAME 레코드**만** 추가
- MX 레코드는 **절대 삭제/수정 금지**

### Q2: 0to1tax.com (www 없이)도 작동하나요?
**A:** 아직은 안 돼요. 먼저 www만 연결하고, 나중에 루트 도메인도 연결할게요.

**지금은**:
- www.0to1tax.com ✅ 작동
- 0to1tax.com ❌ 아직 (나중에 추가)

### Q3: 배포 URL을 어디서 찾나요?
**A:** Publish 탭에서 확인하세요!

**찾는 방법**:
1. 화면 왼쪽 또는 상단 **"Publish"** 클릭
2. **"Deployed"** 또는 **"Live"** 섹션 확인
3. URL 복사

### Q4: https가 안 돼요 (SSL 오류)
**A:** 초반에는 http만 작동할 수 있어요.

**해결책**:
- Cloudflare 사용 (무료 SSL 자동)
- 또는 SSL 인증서 별도 구매

### Q5: 루트 도메인 (www 없이)도 연결하고 싶어요
**A:** 먼저 www 연결 완료 후, A 레코드 추가하면 돼요.

## 🔧 루트 도메인도 연결하기 (선택사항)

**www.0to1tax.com이 작동한 후**에 진행하세요!

### 필요한 정보: IP 주소

**IP 주소 확인 방법**:

#### Windows:
1. **명령 프롬프트** 열기 (Win + R → cmd)
2. 입력:
   ```
   ping [배포된URL에서 https:// 빼고]
   ```
   예: `ping iadkczem.gensparkspace.com`
3. IP 주소 확인 (예: `104.26.15.183`)

#### Mac:
1. **터미널** 열기
2. 입력:
   ```
   ping [배포된URL에서 https:// 빼고]
   ```
3. IP 주소 확인

### A 레코드 추가:

Squarespace DNS 설정:
```
Type: A
Host: @
Points to: [확인한 IP 주소]
TTL: Auto 또는 3600
```

**저장** 후 15분~2시간 대기
